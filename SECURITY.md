# Security

This project is an unofficial Electron client that loads the official ChatGPT website. The security model is designed to keep remote website content isolated from local Node.js and operating-system privileges while adding native Linux desktop integration.

## Remote Content Isolation

The ChatGPT page is remote content. It must not receive unrestricted Node.js access.

Current defaults:

- `nodeIntegration` is disabled for remote content.
- `contextIsolation` is enabled.
- Renderer sandboxing is enabled where technically possible.
- Preload scripts expose only narrow, explicit APIs.
- Local settings and offline pages are served from bundled application files, not from the ChatGPT website.

Do not expose generic filesystem, process, shell, or Electron APIs to remote pages.

## Navigation Policy

Navigation is controlled centrally.

Allowed in-app navigation is limited to trusted ChatGPT and OpenAI hosts needed for normal use and authentication. Ordinary external HTTPS links are opened in the user's default browser.

Blocked navigation includes:

- `javascript:`
- `file:`
- unexpected `data:` URLs
- unknown custom schemes
- untrusted arbitrary sites attempting to take over the main app window

Blocked-navigation logs are sanitized so sensitive query parameters are not recorded.

## Permission Handling

Electron permission requests are handled explicitly.

Supported permissions are granted only to trusted origins and only when needed for ChatGPT features. Unsupported or unnecessary permissions are denied by default, including sensitive device APIs such as geolocation, MIDI, USB, and serial.

Remembered permission decisions can be cleared from Settings.

## Local Settings Storage

Settings are stored as JSON in Electron's user-data directory.

The settings manager:

- Merges missing values with defaults.
- Validates loaded values.
- Recovers safely from malformed JSON.
- Writes settings atomically where possible.
- Rejects unsafe ChatGPT URLs.
- Allows HTTPS ChatGPT/OpenAI hosts by default.
- Never executes data from the settings file.

Settings must not contain ChatGPT credentials, cookies, tokens, or conversation content.

## Profile Isolation

Profiles use separate Electron persistent session partitions:

```text
persist:chatgpt-default
persist:chatgpt-personal
persist:chatgpt-work
```

Profile names are validated to prevent path traversal and unsafe partition names. Cookies, cache, local storage, authentication state, and permission decisions are isolated per profile.

The app must never intentionally copy authentication cookies between profiles.

## Diagnostics Export

Diagnostics export is sanitized before writing to disk.

Allowed diagnostics include application version, Electron/Chromium/Node versions, Linux distribution details, architecture, desktop session information, relevant flags, GPU state, sanitized settings, and sanitized logs.

Diagnostics must not include:

- Cookies
- Authentication tokens
- Chat content
- Uploaded documents
- Downloaded files
- Browser local storage
- Full sensitive URLs
- Personally identifying environment variables

## Data The Application Does Not Collect

The app does not intentionally collect or store:

- ChatGPT credentials
- ChatGPT authentication tokens
- ChatGPT cookies outside Electron's isolated session storage
- Conversation text
- Uploaded documents
- Downloaded files
- Browser local storage outside Electron's profile storage
- Analytics or telemetry generated by this wrapper

The official ChatGPT website may process data according to OpenAI's own service behavior and policies.

## Security Maintenance Rules

- Keep remote content isolated.
- Keep permission grants origin-scoped.
- Keep navigation decisions centralized.
- Redact logs and diagnostics.
- Do not add broad IPC channels.
- Do not add a general-purpose browser mode.
- Do not store secrets in app settings.
